# 绝区零版本时间轴 — 设计标准文件

## 一、项目信息

- 项目名称: 绝区零版本时间轴 (Zenless Zone Zero Version Timeline)
- 项目类型: 静态展示网站（单页面）
- 技术栈: HTML5 + CSS3 + Vanilla JavaScript
- 工作目录: `E:\zzz-timeline\`
- 设计风格: ASCII / 终端终端风格
- 最后更新: 2026/06/25


## 二、文件结构

```
E:\zzz-timeline\
├── DESIGN.md           本文件，设计标准文档
├── index.html          主页面，含完整时间轴数据与结构
├── css\
│   └── style.css       全局样式，ASCII 终端视觉
└── js\
    └── main.js         滚动交互、Intersection Observer 动效
```

项目无外部依赖。所有资源本地自包含。不使用图片、webfont、CDN 或第三方脚本。


## 三、设计规范

### 3.1 设计关键词

- 终端 (Terminal)
- 单色向 (Monochromatic)
- 信息密集 (Information-first)
- 克制 (Restrained)

### 3.2 设计约束

- 不使用 emoji 作为图标、装饰或内容
- 不使用 CSS `content` 插入装饰性字符
- 不使用 `text-shadow` 制作发光效果
- 不使用 `<marquee>` 或 `blink` 等废弃标签
- 不使用 rainbow / gradient text 花字
- 不使用 transform 做过度夸张的动画

### 3.3 色彩方案

```
+---------------+-----------+--------------------------------+
| 用途          | 色值      | 说明                           |
+---------------+-----------+--------------------------------+
| 页面背景      | #0a0a0a   | 接近纯黑，模拟终端底色          |
| 卡片背景      | #111111   | 微高于背景的暗色                |
| 卡片边框      | #2a2a2a   | 细线，终端分隔感                |
| 强调色 (绿)   | #33ff66   | 时间轴线、重点标记、高亮        |
| 辅助色 (琥珀) | #ccaa00   | 版本号、次强调                 |
| 正文颜色      | #b0b0b0   | 柔和灰绿，终端标准亮度          |
| 辅助文字      | #666666   | 日期、次要标注                 |
| 标题文字      | #e0e0e0   | 接近白色，标题专属              |
+---------------+-----------+--------------------------------+
```

注: 不使用半透明色 (RGBA)。全部使用不透明十六进制色值，保持终端纯粹感。

### 3.4 字体

```
+-------------+------------------------------------------------------+
| 用途        | 字体栈                                               |
+-------------+------------------------------------------------------+
| 全局        | "Courier New", "Consolas", "Menlo",                  |
|             | "Source Code Pro", monospace                         |
| 中文回退    | "Microsoft YaHei", "PingFang SC",                    |
|             | "Noto Sans SC", monospace                            |
+-------------+------------------------------------------------------+
```

字体大小阶梯 (rem):

```
12px  = 0.75rem   辅助标注、页脚
13px  = 0.8125rem 次要信息
14px  = 0.875rem  正文
16px  = 1rem      卡片内标题
18px  = 1.125rem  版本号
20px  = 1.25rem   分区标题
32px  = 2rem      页面主标题
```

字重:

- 400: 正文、辅助文字
- 700: 标题、版本号、分区名

行高:

- 正文: 1.6
- 标题: 1.3
- 卡片内: 1.5

### 3.5 间距系统

基于 4px 倍数:

```
4px  / 8px / 12px / 16px / 20px / 24px / 32px / 48px / 64px / 80px
```

### 3.6 边框与分隔

- 卡片: `1px solid #2a2a2a`
- 时间轴竖线: 使用 CSS `width: 2px; background: #33ff66` (不透明度透过色值本身控制，选用 #1a3a1a 近似 40% 绿在黑底上的等效)
- 版本节点圆点: `8px x 8px solid #33ff66`，无边框
- 横线分隔 (分区之间): `border-bottom: 1px dashed #2a2a2a`

### 3.7 ASCII 风格特定规范

- 页面主标题使用 ASCII 标题框格式输出（如用等宽字符拼出边框）
- 版本号前不加 "#" 也不加 "v" 风格化前缀（直接 v1.0 格式）
- 分区标题使用大写字母和等号分隔符样式
- 滚动条若无浏览器默认样式，保持极简（细、浅色）

## 五、视觉呈现

### 5.1 整体布局

```
+--[HEADER]--+
|            |  -- 固定顶部，半透明深色背景，网站标题
+------------+

     |
     |  -- 时间轴竖线 (2px, 绿色, 居中偏左)
     O  -- 版本节点 (8px 圆点)
+---------+
| v1.0    |  -- 版本卡片 (右对齐于时间轴线)
| Title   |
| Detail  |
+---------+
     |
     O
+---------+
| v1.1    |
| ...     |
+---------+
     |
    ...

+--[FOOTER]--+
|            |  -- 底部版权信息
+------------+
```

### 5.2 桌面端布局 (>768px)

- 最大内容宽度: 720px，水平居中
- 时间轴竖线位于左 48px 处
- 版本卡片左边距 80px (线与卡片之间 32px 间隙)
- 卡片内 padding: 20px 24px
- 顶栏高度: 56px
- 第一个卡片距顶栏: 96px

### 5.3 移动端布局 (<768px)

- 时间轴竖线位于左 24px 处
- 版本卡片左边距 52px
- 卡片内 padding: 16px 16px
- 页面主标题字号缩小至 1.5rem
- 分区标题缩小至 1rem

### 5.4 版本卡片结构

```
+--[卡片]------------------------+
| v1.0                    07/04 |  <- 版本号(琥珀) | 靠右日期(灰色)
| 欢迎来到新艾利都               |  <- 中文标题 (白色)
| Welcome To New Eridu           |  <- 英文标题 (灰色)
|                                |
| 全平台公测 PC/iOS/Android/PS5  |  <- 要点列表 (正文灰绿)
| 序章至第三章主线开放            |
+--------------------------------+
```

### 5.5 滚动动画

- 卡片默认 `opacity: 0; transform: translateY(16px)`
- 卡片进入视口时过渡至 `opacity: 1; transform: translateY(0)`
- 使用 Intersection Observer，threshold: 0.15
- 过渡: `0.5s ease-out`
- 无 parallax、无 scroll-jacking

### 5.6 分区标识

三个分区 (测试期 / Season 1 / Season 2 / Season 3) 之间以一行文本标识：

```
==================== SEASON 1 ====================
```

样式: 灰色文本，等宽字符，居中。不使用额外装饰。


## 六、代码规范

### 6.1 HTML

- 使用语义化标签: `<header>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<time>`
- 每个版本卡片以 `<article>` 包裹
- 日期使用 `<time datetime="YYYY-MM-DD">` 标签
- 版本号使用 `<span class="ver">` 标签
- 页面语言: `<html lang="zh-CN">`

### 6.2 CSS

- 所有颜色通过 CSS 自定义属性 (`:root { }`) 集中管理于 `style.css` 顶部
- BEM 命名风格 (Block__Element--Modifier)
- 布局使用 Flexbox
- 仅一处媒体查询断点: `@media (max-width: 768px)`
- 不使用 `!important`
- 不使用 `@import`
- 不使用 CSS 动画关键帧编写复杂动画（仅 transition）
- 不使用 `position: fixed` 以外的 sticky（仅 header 需要）

### 6.3 JavaScript

- `"use strict"` 严格模式
- 使用原生 DOM API（无 jQuery、无框架）
- Intersection Observer API 处理卡片进入视口的淡入动画
- 事件监听使用 `{ passive: true }` 优化滚动性能
- 全局命名空间约束: 单个入口函数 `initTimeline()`
- 变量命名: camelCase
- 常量: UPPER_SNAKE_CASE

### 6.4 性能

- 页面总大小 < 30KB（含 HTML/CSS/JS，未压缩）
- 无外部资源加载（不含 webfont、CDN、图片、第三方脚本）
- 无 Google Analytics、社交媒体嵌入、评论区等第三方组件
- CSS 置于 `<head>`，JS 使用 `<script defer>` 延后执行

### 6.5 可访问性

- 颜色对比度满足 WCAG AA 级别
- 页面有明确 `<title>` 和 `<meta description>`
- 语义标签帮助屏幕阅读器理解页面结构
